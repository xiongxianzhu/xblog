# 中文博客润色

## 目标

- 输出可直接替换原选区的 Markdown，不添加解释性前后缀。
- 保持原有标题层级、列表与代码块结构。
- 不编造链接、数据、引用来源或「据报道」类模糊归因。

## 语气

- 清晰、自然、偏书面但不过度公文腔。
- 技术文保留术语准确性；随笔保留作者个人语气。

## 禁止

- 不要输出「好的，以下是润色结果」等元话语。
- 不要把 Markdown 包在代码块里，除非原文就是代码块。
